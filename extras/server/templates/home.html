✧ What Is a Substrait? ✧

A substrait is a geometric shape whose contours, angles, and density already carry the data. It isn’t a file; it’s a dimensional map you can sample. Change the lens and you read the same shape in color, tone, motion, or numbers.

Live 3D substrait

Drag to rotate. Hover for coordinates, slope, and vector. Generated on the fly; no hardcoded mesh.

Every point in the substrait already carries values: position, slope, angle, pitch, distance from zero, distance from its neighbors. A lens simply remaps that inherent structure into a form we care about, including color, tone, motion, and vectors, so a tiny expression can stand in for a massive dataset (like how e = mc2 encodes tremendous structure without enumerating the universe).

If your browser blocks WebGL, imagine the saddle (z = x·y) and bowl-saddle hybrid (z = x·y²) as layered contours; the data stays in the shape.

What is a lens?

A lens is a deterministic transform from the same underlying shape into a specific signal or view. Swap the lens, keep the shape.

• Inputs: position (x,y,z,m), slope, curvature, density, entropy
• Outputs: color, sound, vector fields, text labels, mesh geometry, events, envelopes

Examples: color lens (height → hue), sound lens (position → pitch), vector lens (gradient → arrows), event lens (inflection → triggers), mesh lens (heightfield → normals/depth).

Lens definitions live right here—before you try the demo—so every reading you see is computed, not staged.

One Law of Dimensions

The rule: Point in higher dim = spans the whole lower dim. Simple addressing, no deep tree to walk.

The creative spiral mirrors this: many steps, one simple address up top.

Why it matters for storage

Addressing is one stride: idx = x + X·(y + Y·(z + Z·m)). A higher-dimensional point can represent an entire lower-dimensional span, so you avoid nested traversal and only materialize data when needed.

Demo tie-in: In the calculator above, increasing M grows capacity without adding lookup depth—access stays O(1); packing is only kept when it beats raw (margin-aware).

Fibonacci spiral layers of creation & dimensional model

Eight beats: void → point → line → width → plane → cube → interlaced cubes → single 4D address. The Fibonacci squares and dimension model evolve together.

0D = void • 1D = x • 2D = xy • 3D = xyz • 4D = xyzm
Pick a step to place its square.

Creative stack (cumulative)

  • 1 Spark: birth point; holds nothing yet.
  • 2 Divide: place the point on x; retains spark.
  • 3 Organize: extend to xy; holds spark + line.
  • 4 Identify: area; holds spark + line + plane.
  • 5 Assemble: layer of cube (xyz); keeps all prior.
  • 6 Human: address xyzm; references every layer beneath.
  • 7 Rest: one 4D point naming the whole cube; all info retained.
